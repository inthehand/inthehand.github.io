Remarks

Note: The address should be supplied in little-endian order on the current Windows platform (which is little-endian). For forward compatibility it would be safer to use the Parse(String) method, which will be correct for all platforms. Or consider CreateFromLittleEndian(Byte) or CreateFromBigEndian(Byte).
